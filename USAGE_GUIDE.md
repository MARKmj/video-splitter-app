# 视频分段工具 - 使用指南

## 🎬 应用介绍

这是一个功能强大的桌面应用程序，用于批量将长视频分割成多个短视频片段。无论您是内容创作者、视频编辑师，还是需要处理大量视频文件的用户，这个工具都能大大提高您的工作效率。

## ✨ 主要特性

- 🚀 **快速处理**: 使用 FFmpeg 硬件加速，无需重新编码
- 🎨 **精美界面**: 现代化设计，渐变紫色主题
- 📊 **实时进度**: 清晰显示处理进度和状态
- 🎯 **精确控制**: 支持自定义分段时长（10-600秒）
- 📦 **批量处理**: 一次性处理多个视频文件
- 🎬 **多格式支持**: MP4, AVI, MOV, MKV, FLV, WMV, WEBM

## 📋 系统要求

- **操作系统**: macOS 10.13+, Windows 10+, 或 Linux
- **Node.js**: 14.0 或更高版本
- **内存**: 至少 4GB RAM
- **磁盘空间**: 根据视频大小而定

## 🚀 快速开始

### 步骤 1: 安装依赖

```bash
cd video-splitter-app
npm install
```

如果下载速度慢，可以使用国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

### 步骤 2: 启动应用

**开发模式**（推荐）:
```bash
npm run dev
```

**生产模式**:
```bash
npm run build
npm start
```

### 步骤 3: 使用应用

1. **选择输入文件夹**
   - 点击"选择文件夹"按钮
   - 选择包含视频文件的文件夹
   - 应用会自动扫描并显示所有视频文件

2. **选择输出文件夹**
   - 点击第二个"选择文件夹"按钮
   - 选择分段后视频的保存位置

3. **设置分段时长**
   - 方法1: 拖动滑块（10-600秒）
   - 方法2: 直接在输入框中输入数字
   - 方法3: 点击预设按钮（30秒、60秒、120秒、180秒、300秒）

4. **选择要处理的视频**
   - 点击"全选"按钮选择所有视频
   - 或单独点击每个视频进行选择
   - 已选择的视频会显示"待处理"标签

5. **开始处理**
   - 点击"开始分段"按钮
   - 观察实时进度显示
   - 等待处理完成

## 🎯 使用示例

### 示例 1: 将长视频分割为 1 分钟片段

```
输入: 一个 60 分钟的视频
设置: 分段时长 = 60 秒
输出: 60 个 1 分钟的视频片段
```

### 示例 2: 批量处理多个视频

```
输入: 文件夹中的 10 个视频文件
设置: 分段时长 = 120 秒
选择: 全选所有视频
输出: 每个视频被分割成 2 分钟的片段
```

## 📂 输出文件命名规则

分段后的文件会自动命名，格式如下：

```
原文件名_01.扩展名
原文件名_02.扩展名
原文件名_03.扩展名
...
```

**示例**:
- 原文件: `我的视频.mp4`
- 输出: `我的视频_01.mp4`, `我的视频_02.mp4`, `我的视频_03.mp4`...

## 💡 使用技巧

### 1. 快速设置常用时长
点击预设按钮可以快速设置常用的分段时长：
- **30秒**: 适合短视频平台
- **60秒**: 适合社交媒体
- **120秒**: 适合教学视频
- **180秒**: 适合中等长度视频
- **300秒**: 适合长视频分割

### 2. 处理大文件
处理大文件时：
- 确保有足够的磁盘空间
- 关闭其他占用内存的程序
- 耐心等待处理完成

### 3. 批量处理建议
- 一次处理不要超过 20 个视频
- 确保所有视频格式相同
- 先用小文件测试设置

## 🔧 高级功能

除了预设按钮，你还可以：

1. **磁盘空间**: 确保输出文件夹有足够空间
2. **文件权限**: 确保对输入输出文件夹有读写权限

4. **备份建议**: 处理重要视频前建议先备份

## 🐛 常见问题

### Q: 为什么有些视频无法处理？
A: 检查视频文件是否损坏，或是否为支持的格式。

### Q: 处理速度很慢怎么办？
A:
- 使用 `-c copy` 模式（默认）
- 关闭其他程序
- 检查磁盘读写速度

### Q: 应用无法启动？
A:
- 检查 Node.js 版本
- 重新安装依赖: `rm -rf node_modules && npm install`
- 查看错误日志

### Q: 分段后的视频画质下降？
A: 正常情况不会，因为我们使用 `-c copy` 不重新编码。如果出现问题，可能是原视频问题。
